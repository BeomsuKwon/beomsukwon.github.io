Docker 기본

0. 설치

Docker CE 설치법(ubuntu)

이전 버전 삭제하기

$ sudo apt-get remove docker docker-engine docker.io containerd runc

apt 저장소 지정

  1. apt 패키지 업데이트

    $ sudo apt-get update

  2. HTTPS를 사용하기 위한 패키지 설치

    $ sudo apt-get install \\
    apt-transport https \\
    ca-certificates \\
    curl \\
    gnupg-agent \\
    software-properties-common

  3. 도커 공식 GPG key 추가

    $ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

    8 글자의 fingerprint 9DC8 5822 9FC7 DD38 854A E2D8 8D81 803C 0EBF CD88로 인증이 허가된다.

    $ sudo apt-key fingerprint 0EBFCD88

pub        rsa4096 2017-02-22 [SCEA]
        9DC8 5822 9FC7 DD38 854A E2D8 8D81 803C 0EBF CD88
uid            [unknown] Docker Release (CE deb) <docker@docker.com>
sub   rsa4096 2017-02-22 [S]

  4. 아래의 커맨드를 실행해 stable 저장소를 세팅한다. nightly나 test저장소를 추가하려면, 아래의 커맨드 끝에 stable이나 test를 붙이면 된다(혹은 둘다)

    $ sudo add-apt-repository \\
   "deb [arch=amd64] <https://download.docker.com/linux/ubuntu> \\
   $(lsb_release -cs) \\
   stable"

도커 CE 설치

  1. apt 업데이트

    $ sudo apt-get update

  2. 최신 버전 설치(docker-ce, docker-cli, containerd.io)

    $ sudo apt-get install docker-ce docker-ce-cli containerd.io

  3. 특정 버전 설치
    a. 설치 가능한 버전을 확인
    b. 두번째 열에 적혀있는 버전5:18.09.1~3-0~ubuntu-xenial을 지정하여 특정 버전을 설치

    $ apt-cache madison docker-ce

  docker-ce | 5:18.09.1~3-0~ubuntu-xenial | <https://download.docker.com/linux/ubuntu>  xenial/stable amd64 Packages
  docker-ce | 5:18.09.0~3-0~ubuntu-xenial | <https://download.docker.com/linux/ubuntu>  xenial/stable amd64 Packages
  docker-ce | 18.06.1~ce~3-0~ubuntu       | <https://download.docker.com/linux/ubuntu>  xenial/stable amd64 Packages
  docker-ce | 18.06.0~ce~3-0~ubuntu       | <https://download.docker.com/linux/ubuntu>  xenial/stable amd64 Packages
  ..

$ sudo apt-get install docker-ce=<VERSION_STRING> docker-ce-cli=<VERSION_STRING> containerd.io

  4. hello-world이미지를 실행시켜 설치 확인

    $ sudo docker run hello-world

1. 기본 문법

search 명령으로 이미지 검색하기

docker search <이미지 이름>

  • Docker Hub의 이미지를 검색한다

sudo 입력하지 않기

루트 계정으로 로그인

$ sudo su

현재 계정을 docker 그룹에 포함(docker 그룹은 root 권한과 동일)

$ sudo usermod -aG docker ${USER}
$ sudo service docker restart

pull 명령으로 이미지 받기

docker pull <이미지 이름>:<태그>

  • Docker Hub에서 우분투 이미지 받아보기

    $ sudo docker pull ubuntu:lastest

  • 이미지 이름 뒤에 lastest를 설정하면 최신 버전을 받음

images명령으로 이미지 목록 출력하기

docker images

  • 모든 이미지를 출력해보기

    $ sudo docker images

run명령으로 컨테이너 생성하기

docker run <옵션> <이미지 이름> <실행할 파일>

  • 이미지를 컨테이너로 생성한 뒤 Bash 셸 실행해보기

    $ sudo docker run -i -t --name hello ubuntu /bin/bash

ubuntu 이미지를 컨테이너로 생성한 뒤 ubuntu 이미지 안의 /bin/bash를 실행

  • -i (interactive), -t (pseudo-tty) 옵션으로 실행된 bash 셸에 입력 및 출력 가능
  • –name 옵션으로 컨테이너에 이름 지정 가능
